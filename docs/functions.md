# 검색
## 룰
* 기본앱도 모두 검색될 것
* 검색어 입력 시 실시간으로 검색됨
* 검색창이 비었을 때는 최근 사용 앱을 표시 (런처에서 실행한 기록 기준)
* 검색결과는 'match 일치', 'history 이력', 'similar 유사', 'related 관련' 섹션으로 분류됨
* 섹션은 위 순서대로 표시되며, 섹션별 표시 개수 제한은 없음
* 한 앱이 여러 섹션 조건에 해당하면 가장 상위 섹션 한 곳에만 표시
* 각 섹션 내부에서는 검색어와의 일치도순으로 정렬

## 검색방식
### match 일치
* 검색어가 앱 정보(앱 이름, 패키지명)에 부분 포함됨
* 한글 초성검색 (앱 이름 대상)

### history 이력
* 검색어로 과거에 similar, related 섹션에서 선택되어 실행된 적이 있음
* 검색창에 입력했던 검색어가 키워드가 되어 다음 검색에서 나타나는 개념
* 이력은 영구 저장됨
* 현재 검색어가 이력 키워드에 부분 포함되면 노출 (match와 동일한 일치 로직)
* match 섹션에서 선택·실행한 경우는 이력에 남기지 않음 (다음 검색에서도 match에 나타나므로)

### similar 유사
* 영문 ↔ 한글 발음검색 (규칙 기반)
* 로마자 변환으로 1차 비교, 일치하지 않으면 음운 골격(자음만 남긴 형태)으로 퍼지 매칭
* 'instagram'과 '인스타그램'처럼 영어 철자와 한글 표기가 같은 골격으로 묶임
* 패키지명(도메인) 검색 — 'google' 입력 시 com.google.* 패키지 전부

### related 관련
* 연속되는 글자수가 가장 일치하는 앱
* 연속되지 않더라도 출현 글자가 가장 일치하는 앱
* 한글은 음절 단위로 비교, 초성만 입력한 경우는 자모 단위로 비교
* 비교 대상은 앱 이름과 패키지명
* 검색어가 2글자 이상일 때만 표시

# 앱관리
* 앱 롱프레스 시 Delete, App info, Play Store 버튼 팝업
* Delete: 일반 앱은 삭제, 시스템(기본)앱은 삭제 불가하여 버튼 비활성(회색) 처리
