# 앱 아이콘 렌더링

런처가 각 앱 아이콘을 어떻게 가져와 표시하는지, 그리고 모양이 제각각으로
보이는 이유를 기록한다. **현재 동작은 제작자가 의도한 형태이며, 통일 마스킹이나
원본 복원 같은 추가 가공은 일부러 하지 않는다.**

## 파이프라인

- 네이티브: [MainActivity.kt](../android/app/src/main/kotlin/com/onethelab/searchitlauncher/MainActivity.kt)
  - `getIcons` → `pm.getApplicationIcon(pkg)`로 각 앱의 아이콘 Drawable을 받음
  - `drawableToPng()`로 **192×192 ARGB_8888 PNG**로 변환 (quality 100, 무손실)
  - 디스크 캐시: `cacheDir/icons_192/<pkg>_<versionCode>.png`
    - 앱 업데이트(`PACKAGE_REPLACED`) 시 해당 캐시 무효화 후 재생성
    - 목록에 없는 패키지 캐시는 prune
- Flutter는 이 PNG 바이트를 그대로 표시 (가공 없음)

### drawableToPng의 두 분기

```kotlin
val bitmap = if (drawable is BitmapDrawable && drawable.bitmap != null) {
    Bitmap.createScaledBitmap(drawable.bitmap, 192, 192, true)  // 레거시: 리사이즈만
} else {
    // 어댑티브/벡터 등: 192 정사각 캔버스에 draw()
    drawable.setBounds(0, 0, 192, 192); drawable.draw(canvas)
}
```

## 모양이 제각각인 이유

아이콘 종류에 따라 위 두 분기로 갈리고, **런처는 통일용 마스크를 적용하지 않기 때문**이다.

| 아이콘 종류 | 처리 | 결과 모양 |
|---|---|---|
| 레거시 (`BitmapDrawable`) | 192px 리사이즈만, 마스크 없음 | 앱이 PNG에 직접 그린 모양 그대로 (사각형·자체 라운딩·원형 등 제각각) |
| 어댑티브 (`AdaptiveIconDrawable`, API 26+) | `draw()` 시 **기기 시스템 기본 마스크**(`config_icon_mask`)로 모서리 클리핑 | 기기 마스크 모양(원/스퀘어클/둥근사각). 기기·OEM마다 다름 |

즉 **OS가 통일성을 위해 마스킹한 게 아니라**, ① 레거시는 앱이 그린 모양 그대로 통과 +
② 어댑티브만 자체 마스크로 모양이 잡히는 것의 혼합이라 일관성이 없어 보인다.
(픽셀·삼성 One UI 등 시스템 런처는 레거시까지 강제로 배경에 얹고 동일 마스크로
깎아 통일하지만, 이 런처는 그 처리를 하지 않는다.)

## 우리가 가감하는 것 / 안 하는 것

**가감 O (런처가 손대는 부분)**
1. **192×192 정사각 리사이즈** — 모든 아이콘 공통. 작은 원본은 확대로 약간 흐려질 수
   있고, 비정사각형 원본은 늘어남(드묾). PNG 압축 자체는 무손실.
2. **어댑티브 아이콘 모서리 잘림** — full bounds로 `draw()`하면 `AdaptiveIconDrawable`이
   시스템 기본 마스크로 바깥을 깎음. 원본 전경/배경 레이어를 통째로 보여주는 게 아니라
   잘린 결과. 안전영역(108dp 중 72dp) 규격상 로고가 레거시보다 작게 보임.

**가감 X (일부러 안 하는 것)**
- 강제 통일 마스크 없음 (레거시를 특정 모양으로 깎지 않음)
- 레거시 아이콘에 배경 합성·여백 추가 없음
- Material You 테마(모노크롬) 틴트·색 변경 없음 — 표준 아이콘 그대로
- 회전·그림자·둥근모서리 덧입히기 없음

## 설계 결정 (의도)

- **레거시 아이콘**: 원본 그대로(리사이즈만). 앱 제작자가 설정한 모양을 존중.
- **어댑티브 아이콘**: 원래 마스킹되도록 설계된 것을 시스템 의도대로 마스킹해 표시.
- 결과적으로 모양이 섞여 보이는 것은 **수용한 트레이드오프**다. 전체 통일 마스킹도,
  어댑티브 원본(풀블리드) 복원도 **하지 않는다.** 이것이 제작자가 원하는 형태다.

> 향후 방향을 바꾼다면 두 갈래뿐이다:
> (a) 어댑티브도 잘리지 않은 전체 레이어(정사각 풀블리드)로 표시, 또는
> (b) 레거시 포함 전부 한 모양으로 통일(배경 합성 + 단일 마스크).
> 둘 다 현재는 채택하지 않음.
